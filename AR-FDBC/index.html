非遗：福鼎白茶制作技艺

一款基于 Unity AR 技术打造的项目，通过识别特定图像，让用户沉浸式体验福鼎白茶产区介绍、采摘到品饮的制作流程及相关传说，还可借助 AI 答疑，助力非遗茶文化传播。

项目概览

项目简介

本项目是一款以 “福鼎白茶非遗文化传承” 为核心的 Unity AR 交互作品，通过虚实融合技术将福鼎白茶的地理产区、制作工艺与文化传说转化为沉浸式体验。

项目以 5 张专属识别图为交互入口，构建了从 “认知产区” 到 “体验工艺” 再到 “感知文化” 的完整内容链路，既还原了福鼎白茶 “自然萎凋、不炒不揉” 的非遗制作技艺，又通过音频讲解、三维模型、视频资料等多媒体形式，传递 “一年茶、三年药、七年宝” 的茶文化内涵。

右侧为五张识别图，分别对应不同的内容模块，用户通过识别图像即可触发相关的 AR 场景与交互体验。

游戏玩法

以 “识别图触发 + 场景交互” 为核心玩法，操作逻辑清晰且门槛低

目标受众

青少年群体，茶文化爱好者与茶行业从业者，文旅体验者与游客，教育机构与非遗保护单位

项目愿景

打破传统图文传播的局限，以 AR 技术让福鼎白茶的产地风貌、制作工艺、历史传说 “活起来”，助力非遗文化与地方特色产业的传承

创作过程

阶段一

设计思路

项目初始的概念设计与规划阶段

本项目旨在通过 AR 技术实现福鼎白茶文化的沉浸式传播，核心设计思路围绕 “技术赋能文化传播” 展开，以用户交互体验为核心，整合 AR 交互、智能问答与场景化内容呈现。

聚焦福鼎白茶的 “产区认知 - 制作流程 - 文化传说” 三大信息维度，通过 AR 技术打破传统信息传播的平面化限制，让用户通过识别相关图像，触发虚拟三维模型、音视频解说及智能问答，实现 “现实场景 + 虚拟内容” 的深度融合，降低用户获取白茶文化知识的门槛。

阶段二

核心功能开发

基于设计思路，实现项目的核心功能与技术框架

1 AR 交互核心功能开发

三维模型交互系统：独立开发脚本，实现模型的触摸交互功能，支持单点触摸下的三种旋转模式（自由旋转 / 水平旋转 / 上下旋转）及多点触摸缩放，通过变量控制旋转逻辑，并限制缩放范围，优化用户对白茶模型、产区地图的细节观察体验。

场景切换与流程控制：基于 Unity 的SceneManager实现多场景管理，支持首页与 5 个 AR 识别图场景的快速跳转及程序退出功能，保障用户在不同内容模块间的流畅切换。

2 UI 交互与用户体验优化

功能按钮逻辑实现：为场景中的虚拟按钮绑定事件，解析按钮名称，动态切换模型旋转模式，并重置模型角度，降低用户操作学习成本。

输入输出交互设计：关联 UI 组件，在用户提交问题后清空输入框并显示加载状态提示，提升交互反馈清晰度。

3 多媒体内容整合与 AR 场景构建

识别图触发逻辑：基于 Unity AR 技术，将 5 张识别图与对应虚拟内容（关联，实现现实场景与虚拟信息的精准叠加。

基于Chaos物理系统，实现了真实的物体物理模拟，包括刚体碰撞、约束和布料模拟。优化了物理性能，确保复杂场景下的稳定运行。

多模态内容呈现：整合三维模型、音频解说、工艺演示视频等资源，通过 AR 交互串联 “产区介绍 - 制作流程 - 文化传说” 完整内容链条，打造沉浸式信息获取体验。

阶段三

技术难点与解决方案

开发过程中遇到的挑战及应对策略

AI 智能问答模块集成

第三方 API 对接：通过 UnityWebRequest 实现与 DeepSeek 大语言模型的 HTTP 通信，封装请求头与请求参数，支持用户输入文本的实时提交。

数据解析与响应处理：定义DeepSeekResponse、Choice等序列化类，解析 API 返回的 JSON 数据，提取 AI 回答内容并展示在 UI 文本组件中，同时处理网络请求异常，确保问答功能稳定性。

配置管理：静态类统一管理 API 密钥，实现配置与业务逻辑解耦，便于后期密钥更新与维护。

最终成果

多模式 AR 模型交互系统

精细化触摸交互逻辑，支持单点触摸下的三种旋转模式及多点触摸缩放功能，用户可根据需求灵活切换操作方式

轻量化 AI 问答集成

基于 DeepSeek 大语言模型 API 构建智能问答模块，实现数据序列化 / 反序列化，支持用户实时查询白茶相关知识

模块化场景流程设计

实现多场景的快速切换与退出功能，用户可通过 UI 按钮在 “产区介绍 - 制作流程 - 文化传说” 等内容模块间无缝跳转，确保信息传递的逻辑连贯性
